Key functions to implement:

matchCanonical(docType) — fuzzy-matches a doc type string against CANONICAL_SECTIONS using matchKeys
isKnownStandardType(docType, knownCsiDocTypes) — case-insensitive check if a doc type exists in the known CSI types set
loadKnownCsiDocTypes() — queries the csi_document_requirements table for all known standard document type names, plus adds PROJECT_LEVEL_TYPES. Returns a Set<string>.
buildActiveSections(sortedDocTypes, projectLevelDocTypes, knownCsiDocTypes) — the core logic:
First pass: loops through NON_DIRECTORY_SECTIONS (skips Directory). For each canonical section, tries to match ONLY known standard CSI document types (uses isKnownStandardType check before matchCanonical). Includes the section if it has a matching doc type, OR if it matches a project-level doc type, OR if it's in ALWAYS_INCLUDE_FOLDERS (Permits, Inspections & Sign-Offs).
Second pass: collects all remaining doc types NOT placed in the first pass — these are custom types. Appends each as its own section using its actual name as the folder name.
Final step: renumbers ALL active sections sequentially starting from 2 (Directory is always 1). No gaps.
canonicalFolderName(section) — formats NN-FolderName (e.g., 02-Permits, 07-Equipment O&M's)
sortDocTypes(keys, knownCsiDocTypes) — sorts doc types with known standard types first (in canonical order), then custom types alphabetically
